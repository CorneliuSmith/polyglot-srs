3. Create **backend/services/srs.py** with:
   - `AnswerResult` enum: CORRECT, CORRECT_SLOPPY, WRONG_FORM, WRONG
   - `QUALITY_MAP` dict mapping AnswerResult -> int (4, 3, 2, 1). This is a LOCKED DECISION.
   - `map_answer_to_quality(result: AnswerResult) -> int` convenience function
   - `CardState` dataclass: ease_factor=2.5, interval=1, repetitions=0, streak=0, lapses=0
   - `SRSUpdate` dataclass: ease_factor, interval, repetitions, streak, lapses, next_review (datetime)
   - Constants: EASE_FLOOR=1.3, EASE_RECOVERY_THRESHOLD=5, EASE_RECOVERY_INCREMENT=0.05, EASE_TARGET=2.5
   - `sm2_update(state: CardState, quality: int) -> SRSUpdate` pure function implementing:
     - quality >= 3: success path. repetitions 0->interval 1, repetitions 1->interval 6, else interval=round(interval*ease). Increment repetitions and streak.
     - quality < 3: failure path. Reset repetitions=0, interval=1, streak=0. Increment lapses.
     - Standard SM-2 ease formula: ease + 0.1 - (5-q)*(0.08 + (5-q)*0.02). Floor at 1.3.
     - Ease recovery: if streak >= EASE_RECOVERY_THRESHOLD and ease < EASE_TARGET, add EASE_RECOVERY_INCREMENT (capped at EASE_TARGET).
     - Interval fuzzing: if interval > 2, multiply by random.uniform(0.95, 1.05), ensure >= 1.
     - next_review = datetime.now(timezone.utc) + timedelta(days=interval)
   - Use `datetime.now(timezone.utc)` (NOT deprecated `datetime.utcnow()`).

REFACTOR: Extract constants, ensure docstrings, verify type annotations.
