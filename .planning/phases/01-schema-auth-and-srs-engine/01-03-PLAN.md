Create the core application infrastructure:

1. **backend/repositories/pool.py** -- asyncpg pool lifecycle and RLS context manager:
   - Module-level `_pool: asyncpg.Pool | None = None`
   - `async def init_pool(dsn: str)` -- creates pool with `min_size=2, max_size=10, command_timeout=30`. If `settings.environment != "development"`, also set `statement_cache_size=0` for Supavisor compatibility (Pitfall 3 from research).
   - `async def close_pool()` -- closes pool if initialized.
   - `def get_pool() -> asyncpg.Pool` -- returns pool, asserts not None.
   - `@asynccontextmanager async def rls_connection(user_id: str)` -- acquires connection from pool, sets `request.jwt.claims` with `{"sub": user_id, "role": "authenticated"}` via `set_config(..., true)` (transaction-scoped per Pitfall 5), sets role to `authenticated`, yields connection. CRITICAL: third argument to set_config MUST be `true` for transaction-local scope.

2. **backend/dependencies.py** -- JWT validation FastAPI dependency:
   - `security = HTTPBearer(auto_error=False)`
   - `async def get_current_user(cred: HTTPAuthorizationCredentials = Depends(security)) -> dict` -- validates JWT using `jwt.decode()` with `settings.supabase_jwt_secret`, audience="authenticated", algorithms=["HS256"]. Returns `{"id": payload["sub"], "email": payload.get("email")}`. Raises 401 on missing cred or PyJWTError.
   - Import settings from `backend.config` (not relative import).

3. **backend/main.py** -- FastAPI app:
   - `@asynccontextmanager async def lifespan(app)` -- calls `init_pool(settings.database_url)` on startup, `close_pool()` on shutdown.
   - `app = FastAPI(title="PolyglotSRS", lifespan=lifespan)`
   - CORS middleware with `settings.cors_origins`, `allow_credentials=True`, `allow_methods=["*"]`, `allow_headers=["*"]`.
   - Mount routers: `app.include_router(auth_router, prefix="/api/auth", tags=["auth"])`, `app.include_router(languages_router, prefix="/api/languages", tags=["languages"])`, `app.include_router(review_router, prefix="/api/review", tags=["review"])`.
   - Health check: `GET /api/health` returns `{"status": "ok"}`.

4. **backend/routers/__init__.py** -- empty init file.

5. **backend/tests/test_auth.py** -- JWT validation unit tests using PyJWT to create test tokens:

   - `test_expired_jwt_returns_401` -- create JWT with exp in the past, verify 401.
   - `test_missing_auth_header_returns_401` -- call with no Authorization header, verify 401.
   - `test_invalid_jwt_returns_401` -- call with garbage token, verify 401.

Create the repository functions and API routers:

1. **backend/repositories/languages.py**:
   - `async def get_all_languages(pool: asyncpg.Pool) -> list[dict]` -- no RLS needed (public table). Queries `SELECT id, code, name, rtl FROM languages ORDER BY name`. Returns list of dicts. Uses `pool.fetch()` directly (no rls_connection needed for public tables).

2. **backend/repositories/cards.py**:
   - `async def get_due_cards(conn: asyncpg.Connection, language_id: str, limit: int = 20) -> list[dict]` -- queries user_cards WHERE `language_id = $1 AND next_review <= now() AND is_suspended = false ORDER BY next_review ASC LIMIT $2`. RLS filters by user automatically. Returns list of dicts.
   - `async def update_card_srs(conn: asyncpg.Connection, card_id: str, srs_update: dict) -> None` -- UPDATE user_cards SET ease_factor, interval, repetitions, next_review, last_review=now(), streak, lapses WHERE id = card_id.

3. **backend/repositories/review.py**:
   - `async def insert_review_log(conn: asyncpg.Connection, user_id: str, card_id: str, quality: int, ease_before: float, ease_after: float, interval_before: int, interval_after: int, time_taken_ms: int | None, answer_result: str | None) -> dict` -- INSERT INTO review_log with all fields, RETURNING id and created_at.

4. **backend/routers/auth.py** -- auth router:
   - `router = APIRouter()`
   - `GET /me` -- depends on `get_current_user`, returns the user dict (id, email). This is the "current user" endpoint. No DB call needed -- info comes from JWT.
   - `POST /profile` -- depends on `get_current_user`, creates user_profiles row via rls_connection if not exists. Accepts optional `batch_size`, `ui_language`, `active_language_id`. Returns profile dict. Uses INSERT ... ON CONFLICT (id) DO UPDATE for upsert.
   - `GET /profile` -- depends on `get_current_user`, returns user_profiles row via rls_connection. Returns 404 if no profile exists.

5. **backend/routers/languages.py** -- languages router:
   - `router = APIRouter()`
   - `GET /` -- returns all languages from `get_all_languages(get_pool())`. No auth required (public data, but still useful for language picker).

6. **backend/routers/review.py** -- review router:
   - `router = APIRouter()`
   - `GET /due` -- depends on `get_current_user`. Accepts `language_id` query param (required UUID). Uses `rls_connection(user["id"])` to call `get_due_cards`. Returns list of due cards (SRS-05: sorted by next_review ASC).
   - `POST /submit` -- depends on `get_current_user`. Accepts JSON body: `card_id` (UUID), `answer_result` (string: correct/sloppy/wrong_form/wrong), `time_taken_ms` (int, optional). Inside `rls_connection(user["id"])`:
     1. Fetch current card state from user_cards by card_id.
     2. Map answer_result string to AnswerResult enum, then to quality via `map_answer_to_quality`.
     3. Build CardState from current DB values, call `sm2_update(state, quality)`.
     4. Call `update_card_srs` with the SRSUpdate values.
     5. Call `insert_review_log` with before/after ease, before/after interval, quality, time_taken_ms, answer_result (SRS-04).
     6. Return the SRSUpdate (next_review, new ease, new interval) plus the quality used.
   - All operations in submit happen on the same `conn` from rls_connection (single RLS context).

7. **backend/tests/test_schema.py** -- schema validation test stubs (require running Supabase):
   - `@pytest.mark.skipif` decorated (skip if no DATABASE_URL env var).
   - `test_all_tables_exist` -- connect to DB, query information_schema.tables, assert all 10 tables exist.
   - `test_language_seed_data` -- query languages table, assert ru/ar/en exist with correct RTL flags.
   - `test_language_parameterization` -- query information_schema.columns, assert language_id column exists on grammar_points, vocabulary, content_lists, user_cards.
   - `test_morphology_jsonb` -- assert vocabulary.morphology column type is jsonb.
   - `test_translations_unique_constraint` -- assert UNIQUE on (vocabulary_id, locale).
   - `test_drill_sentences_fk` -- assert drill_sentences.grammar_point_id FK exists.

8. **backend/tests/test_rls.py** -- RLS integration test stubs (require running Supabase with two test users):
   - `@pytest.mark.skipif` decorated (skip if no DATABASE_URL env var).
   - `test_user_cannot_see_other_user_cards` -- using rls_connection with two different user_ids, verify user A cannot SELECT user B's cards.
   - `test_user_cannot_insert_card_for_other_user` -- verify INSERT with mismatched user_id is rejected by RLS.
   - `test_review_log_is_append_only` -- verify UPDATE and DELETE on review_log are blocked by RLS policy.
   - These are stubs with clear TODO comments explaining the test setup needed (two Supabase auth users). They will be fully implemented when a local Supabase instance is available.
