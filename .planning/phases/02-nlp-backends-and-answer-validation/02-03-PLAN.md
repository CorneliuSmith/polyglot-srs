  <behavior>
    ## Normalization (NLP-06)
    - normalize("كَتَبَ") -> "كتب" (strip tashkeel/diacritics)
    - normalize("أكتب") -> "اكتب" (normalize hamza-on-alef to bare alef)
    - normalize("إسلام") -> "اسلام" (normalize hamza-below-alef)
    - normalize("آخر") -> "ااخر" (normalize alef-madda -- note: normalize_alef_ar handles this)
    - normalize("كتـــب") -> "كتب" (strip tatweel/kashida)
    - normalize("  كتب  ") -> "كتب" (strip whitespace)

    ## Diacritic Invariance (NLP-07)
    - check_answer("كتب", "كَتَبَ") -> (CORRECT, None) -- diacritics stripped before comparison
    - check_answer("كَتَبَ", "كتب") -> (CORRECT, None) -- reverse direction also works
    - check_answer("كُتُب", "كَتَبَ") -> should NOT be CORRECT (different vowel patterns = different words)
      Actually after dediac both become "كتب" -- this is expected behavior per spec: diacritics stripped for comparison

    ## Taa Marbuta Handling (research pitfall #6)
    - check_answer with taa marbuta (ة) vs ha (ه) difference should return CORRECT_SLOPPY, not CORRECT
    - Do NOT normalize taa marbuta in the primary normalize() method
    - Add secondary check: if normalize(user) != normalize(correct) but they match after taa-marbuta->ha conversion, return CORRECT_SLOPPY

    ## Lemmatization (NLP-06)
    - lemmatize("كتب") -> returns the lex field from camel-tools analysis (cleaned of sense IDs like _1)
    - If no analysis found, return dediacritized input as fallback

    ## Root Extraction (NLP-06)
    - Extract root from camel-tools analysis dict "root" key
    - Used for verb form comparison in NLP-08

    ## Verb Form Detection (NLP-08)
    - When user answer and correct answer share the same root but have different lemmas/patterns, return WRONG_FORM
    - check_answer("كتب", "كاتب", {"morphology": {"root": "ك.ت.ب"}}) -> (WRONG_FORM, message with root display)
    - Message should display root as spaced consonants: "Root: ك-ت-ب"
    - If card_context has "morphology.verb_form" info, include form number in explanation

    ## Morphological Family (NLP-06)
    - get_morphological_family returns at minimum the lemma
    - Full enumeration via camel-tools Generator is expensive -- return a minimal set (lemma + input word) and rely on the lemma check (Layer 3) for most matching
    - If card_context provides additional known forms, include those

    ## Aspect Partner (NLP-06)
    - get_aspect_partner returns None for Arabic (Arabic verb aspect is handled through verb form detection, not aspect pairs like Russian)

    ## Full Pipeline
    - check_answer("كتب", "كتب") -> (CORRECT, None)
    - check_answer("كَتَبَ", "كتب") -> (CORRECT, None) -- diacritics don't matter
    - check_answer("قرأ", "كتب") -> (WRONG, None) -- completely different word
  </behavior>
  <implementation>
    Create `backend/services/nlp/arabic.py` with class `ArabicNLP(BaseNLP)`:

    1. Module-level initialization:
       ```python
       db = MorphologyDB.builtin_db()
       analyzer = Analyzer(db, backoff='NOAN_PROP')
       ```
       Wrap in try/except -- if camel_data not installed yet, log warning and set analyzer to None.

    2. `normalize(text)`:
       - Strip whitespace
       - `dediac_ar(text)` -- strip all diacritics
       - `normalize_alef_ar(text)` -- normalize alef variants
       - Remove tatweel: `text.replace('\u0640', '')`
       - Do NOT normalize taa marbuta (ة -> ه) here (per research pitfall #6)

    3. `lemmatize(word)`:
       - If analyzer is None, return `dediac_ar(word)`
       - Run `analyzer.analyze(dediac_ar(word))`
       - Sort by `pos_lex_logprob` (highest/least-negative first) to pick most likely analysis
       - Return `lex` field, stripped of sense ID suffix (e.g., "كَتَب_1" -> "كتب")
       - Fallback to dediac input if no analyses

    4. `get_morphological_family(word)`: Return `{word, self.lemmatize(word)}` as minimal set. Camel-tools Generator enumeration is too slow for real-time use.

    5. `get_aspect_partner(verb, card_context=None)`: Return None. Arabic doesn't use Russian-style aspect pairs. MUST match standardized signature: `(self, verb: str, card_context: dict | None = None) -> str | None`.

    6. Override `check_answer()`: After calling `super().check_answer()`, run additional checks REGARDLESS of result (not only when WRONG), and return the enhanced result if a more specific category applies:
       - Check taa marbuta fallback: if only difference between normalized forms is ة vs ه, return (CORRECT_SLOPPY, "Close -- check taa marbuta (ة) vs ha (ه)"). This upgrades a WRONG to CORRECT_SLOPPY, or keeps CORRECT_SLOPPY if already at that level.
       - Check verb form: if both user input and correct answer share the same root (from analyzer or card_context), return (WRONG_FORM, "Wrong verb form -- Root: ك-ت-ب"). This applies when the super() result was WRONG -- same root but different pattern means wrong verb form.
       - If no additional check matches, return the original super() result unchanged.

    CRITICAL: camel-tools requires `camel_data -i light` to be run before the Analyzer works. The test file should skip tests if camel-tools data is not installed. Use `pytest.importorskip("camel_tools")` and check for data availability.

    Install: `pip install camel-tools && camel_data -i light`
    Note: Requires cmake, boost, and Rust compiler as build deps. Python <=3.12.
