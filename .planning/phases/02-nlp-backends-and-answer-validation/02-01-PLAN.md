    1. Add NLP dependencies to pyproject.toml: pymorphy3, camel-tools, spacy, lemminflect, cyrtranslit. Note in a comment that camel-tools requires Python <=3.12 and system deps (cmake, boost).

    2. Create `backend/services/nlp/__init__.py` -- empty for now (registry comes in Task 2).

    3. Create `backend/services/nlp/base.py`:
       - Move `AnswerResult` enum here from srs.py. Keep same values (CORRECT="correct", CORRECT_SLOPPY="correct_sloppy", WRONG_FORM="wrong_form", WRONG="wrong").
       - Define `BaseNLP(ABC)` with abstract methods: `normalize(text: str) -> str`, `lemmatize(word: str) -> str`, `get_morphological_family(word: str) -> set[str]`, `get_aspect_partner(verb: str, card_context: dict | None = None) -> str | None`.
       - CRITICAL SIGNATURE: `get_aspect_partner` takes TWO parameters (plus self): `verb` and `card_context=None`. This is the standardized signature used across ALL plans and backends. The answer-validation-spec.md shows a one-parameter version -- ignore that; use the two-parameter version per this plan.
       - Implement `check_answer(user_input: str, correct_answer: str, card_context: dict | None = None) -> tuple[AnswerResult, str | None]` as a concrete template method with 6 layers:
         Layer 1: Exact match after NFC normalization + strip -> CORRECT
         Layer 2: self.normalize(user) == self.normalize(correct) -> CORRECT
         Layer 3: self.lemmatize(user) == self.lemmatize(correct) -> CORRECT_SLOPPY with message
         Layer 4: self.normalize(user) in {self.normalize(f) for f in self.get_morphological_family(correct)} -> CORRECT_SLOPPY with message
         Layer 5: aspect partner check via `self.get_aspect_partner(correct, card_context)` -> WRONG_FORM with message. MUST pass card_context as second argument.
         Layer 6: card_context["answer_alternatives"] normalized check -> CORRECT
         Default: WRONG, None
       - Apply `unicodedata.normalize('NFC', ...)` as the FIRST step before any comparison (per research pitfall #1).

    4. Update `backend/services/srs.py`:
       - Remove the `AnswerResult` class definition.
       - Add `from backend.services.nlp.base import AnswerResult` at the top.
       - Keep QUALITY_MAP, map_answer_to_quality unchanged -- they now import AnswerResult from nlp.base.

    5. Update `backend/tests/test_nlp_base.py` (created in 02-00) to turn tests GREEN. The StubNLP should have simple, predictable behavior (e.g., normalize = lowercase, lemmatize = strip trailing 's', morphological_family = {word, word+'s'}, get_aspect_partner = lookup from a small dict using card_context). Ensure the stub's get_aspect_partner signature is `(self, verb, card_context=None)`.

    IMPORTANT: Do NOT normalize taa marbuta in the base normalize -- that is language-specific (per research pitfall #6). The base class only applies NFC normalization; each language backend adds its own normalization in its override.
