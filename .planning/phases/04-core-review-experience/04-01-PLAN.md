  <name>Task 1: Extend get_due_cards to JOIN card content and add learn batch + dashboard repositories</name>
  <files>backend/repositories/cards.py, backend/repositories/dashboard.py</files>
  <action>
1. In `backend/repositories/cards.py`, rewrite `get_due_cards()` to JOIN against vocabulary/grammar_points and drill_sentences based on `card_type`:
   - For card_type='vocabulary': JOIN vocabulary ON user_cards.card_id = vocabulary.id, JOIN translations ON vocabulary.id = translations.vocabulary_id WHERE translations.locale = 'en' (default). Return: word, reading, morphology, alternatives, definition (from translations), plus all existing SRS fields. **IMPORTANT: Vocabulary cards do NOT have {{answer}} markers.** The drill mode is "type-the-word": the sentence field is set to the definition/prompt text, and correct_answer = vocabulary.word. The frontend handles this as a separate mode from fill-in-the-blank.
   - For card_type='grammar': JOIN grammar_points ON user_cards.card_id = grammar_points.id, JOIN drill_sentences ON grammar_points.id = drill_sentences.grammar_point_id. Return: sentence (with {{answer}} marker), translation, hint, title, plus SRS fields. The correct_answer must be extracted from the sentence context or stored separately -- for now use grammar_points.title as a placeholder since drill sentence answers are embedded in the sentence pattern.
   - Use a UNION ALL approach or two separate queries merged in Python. Return a list of dicts with unified shape: `{id, card_type, card_id, sentence, correct_answer, hint, translation, morphology, alternatives, language_code, ease_factor, interval, repetitions, streak, lapses, next_review}`.
   - Also add a JOIN to languages table to get the language `code` (needed for NLP backend lookup).

2. Add `add_learn_batch()` function to `backend/repositories/cards.py`:
   - Parameters: conn, user_id, language_id, batch_size
   - Query: SELECT vocabulary.id FROM vocabulary JOIN content_lists ON vocabulary.language_id = content_lists.language_id AND content_lists.list_type = 'vocabulary' JOIN user_content_subscriptions ON content_lists.id = user_content_subscriptions.content_list_id WHERE user_content_subscriptions.user_id = $user_id AND vocabulary.language_id = $language_id AND vocabulary.id NOT IN (SELECT card_id FROM user_cards WHERE user_id = $user_id AND card_type = 'vocabulary') ORDER BY vocabulary.frequency_rank ASC NULLS LAST LIMIT $batch_size
   - INSERT selected vocabulary IDs into user_cards with card_type='vocabulary', next_review=now(), default SRS values
   - Return count of items added and their IDs
   - Note: For Phase 4, only vocabulary learn mode is needed (grammar points get CEFR ordering in Phase 5). If user has no subscriptions, return 0 added with empty list.

3. Create `backend/repositories/dashboard.py`:
   - `get_dashboard_stats(conn, user_id, language_id)` returns dict with:
     - `due_count`: COUNT(*) FROM user_cards WHERE language_id=$1 AND next_review <= now() AND is_suspended = false
     - `streak_days`: Compute from review_log -- count consecutive days (including today) where the user has at least one review for this language. Use: SELECT DISTINCT DATE(created_at) FROM review_log JOIN user_cards ON review_log.card_id = user_cards.id WHERE review_log.user_id = $1 AND user_cards.language_id = $2 ORDER BY DATE(created_at) DESC. Then iterate from today backward counting consecutive days.
     - `cefr_progress`: For each CEFR level (A1-C2), compute fraction of vocabulary items at that level that the user has in user_cards. Query: SELECT v.level, COUNT(uc.id) as learned, COUNT(v.id) as total FROM vocabulary v LEFT JOIN user_cards uc ON v.id = uc.card_id AND uc.card_type = 'vocabulary' AND uc.user_id = $1 WHERE v.language_id = $2 AND v.level IS NOT NULL GROUP BY v.level.
