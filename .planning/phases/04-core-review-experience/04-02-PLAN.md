<task type="auto">
  <name>Task 2: Supabase client, API client, type definitions, Zustand stores, and App shell with routing</name>
  <files>frontend/src/lib/supabase.ts, frontend/src/api/client.ts, frontend/src/api/types.ts, frontend/src/api/review.ts, frontend/src/api/dashboard.ts, frontend/src/api/profile.ts, frontend/src/stores/authStore.ts, frontend/src/stores/prefsStore.ts, frontend/src/App.tsx</files>
  <action>
1. `frontend/src/lib/supabase.ts`:
   - Create Supabase client using `import.meta.env.VITE_SUPABASE_URL` and `import.meta.env.VITE_SUPABASE_ANON_KEY`.
   - Export the `supabase` instance.

2. `frontend/src/api/types.ts` -- TypeScript interfaces for all API contracts:
   ```
   Language { id: string; code: string; name: string; rtl: boolean }
   DueCard { id: string; card_type: 'grammar' | 'vocabulary'; card_id: string; sentence: string; correct_answer: string; hint?: string; translation?: string; morphology: Record<string, unknown>; alternatives: string[]; language_code: string; ease_factor: number; interval: number; repetitions: number; streak: number; lapses: number; next_review: string }
   ValidateAnswerRequest { language_code: string; user_input: string; correct_answer: string; card_context?: Record<string, unknown> }
   ValidateAnswerResponse { answer_result: 'correct' | 'correct_sloppy' | 'wrong_form' | 'wrong'; feedback: string | null }
   SubmitReviewRequest { card_id: string; answer_result: string; time_taken_ms: number | null }
   SubmitReviewResponse { next_review: string; ease_factor: number; interval: number; quality: number }
   LearnResponse { added: number; items: string[] }
   DashboardStats { due_count: number; streak_days: number; cefr_progress: Record<string, number> }
   UserProfile { id: string; batch_size: number; ui_language: string; active_language_id: string | null; created_at: string; updated_at: string }
   ProfileUpdate { batch_size?: number; ui_language?: string; active_language_id?: string }
   ```

3. `frontend/src/api/client.ts`:
   - Create axios instance with baseURL from `import.meta.env.VITE_API_BASE_URL ?? ''` (empty string uses Vite proxy).
   - Request interceptor: call `supabase.auth.getSession()`, attach `Authorization: Bearer ${session.access_token}` if session exists.
   - Response interceptor: on 401, call `supabase.auth.refreshSession()` and retry once.

4. `frontend/src/api/review.ts`:
   - `getDueCards(languageId: string): Promise<DueCard[]>`
   - `validateAnswer(req: ValidateAnswerRequest): Promise<ValidateAnswerResponse>`
   - `submitReview(req: SubmitReviewRequest): Promise<SubmitReviewResponse>`
   - `startLearnSession(languageId: string): Promise<LearnResponse>`
   All using apiClient from client.ts.

5. `frontend/src/api/dashboard.ts`:
   - `getDashboardStats(languageId: string): Promise<DashboardStats>`

6. `frontend/src/api/profile.ts`:
   - `getProfile(): Promise<UserProfile>`
   - `updateProfile(data: ProfileUpdate): Promise<UserProfile>`
   - `getLanguages(): Promise<Language[]>`

7. `frontend/src/stores/authStore.ts`:
   - Zustand store with: `session: Session | null`, `setSession(s)`, `isAuthenticated` (computed getter via get())
   - Do NOT store the access token separately -- get it from session.access_token in the API interceptor.

8. `frontend/src/stores/prefsStore.ts`:
   - Zustand store with persist middleware (localStorage key: 'polyglot-prefs'):
     - `activeLanguageId: string | null`
     - `setActiveLanguageId(id: string)`
   - Persist ensures language selection survives page refresh.

9. `frontend/src/App.tsx`:
   - useEffect to call `supabase.auth.getSession()` on mount and set session in authStore.
   - `supabase.auth.onAuthStateChange` listener to track login/logout/token refresh.
   - Set up react-router-dom with `createBrowserRouter` and `RouterProvider`:
     - `/login` -- placeholder LoginPage (just a div with "Login" text for now)
     - `/` -- placeholder DashboardPage (just a div with "Dashboard" text)
     - `/review` -- placeholder ReviewSessionPage
     - `/learn` -- placeholder LearnPage
   - Protected route wrapper: if not authenticated, redirect to /login.
   - Wrap in `<QueryClientProvider>`.
